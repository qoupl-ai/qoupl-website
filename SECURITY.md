## Disclosure Process

1. We will acknowledge receipt of your report.
2. We will investigate and validate the issue.
3. If accepted, we will work on a fix and coordinate a release.
4. Public disclosure will occur **only after a fix is available**, unless otherwise agreed.

We may request additional information during the investigation.

---

## Automated Security Measures

This repository uses the following security protections:
- GitHub **Dependabot alerts & updates**
- **Secret scanning & push protection**
- **Code scanning** on pull requests
- **Branch protection rules** requiring CI checks

---

## Safe Harbor

We support responsible security research and will not pursue legal action against researchers who:
- Follow this policy
- Avoid data destruction, service disruption, or privacy violations
- Do not exploit issues beyond proof-of-concept

Thank you for helping keep this project and its users secure.
